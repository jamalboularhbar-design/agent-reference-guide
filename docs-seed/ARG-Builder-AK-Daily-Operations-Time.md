# ArtKech Design Studio — Daily Operations & Time Management

## 1. Purpose

A creative studio thrives on structured flexibility — enough routine to maintain productivity and deadlines, enough freedom to allow creative flow. This SOP defines how ArtKech operates day-to-day: studio hours, daily rituals, time tracking, capacity planning, and the rhythm that keeps 8-12 team members aligned and productive.

## 3. Daily Rituals

### 3.1 Morning Standup (09:15, 10 minutes max)

| Element | Format |
| :--- | :--- |
| Who attends | All team members present in studio |
| Format | Standing (literally), round-robin |
| Each person shares | 1) What I'm working on today 2) Any blockers 3) ETA on current deliverable |
| Facilitated by | Studio Coordinator (rotates weekly) |
| Documentation | Key blockers noted in Notion daily log |

### 3.2 Deep Work Blocks

| Block | Time | Rules |
| :--- | :--- | :--- |
| Morning focus | 09:30 – 12:00 | No meetings, no interruptions, headphones = do not disturb |
| Afternoon focus | 14:00 – 16:30 | Same rules apply |
| Communication window | 12:00 – 14:00, 16:30 – 18:00 | Meetings, calls, reviews, collaboration |

### 3.3 End-of-Day Ritual (17:45)

Each team member spends 15 minutes:
- Saving all work in progress to the correct project folder
- Updating time tracking for the day
- Moving completed tasks to "Done" in Notion
- Noting tomorrow's priorities in personal task list
- Committing any code/design files to version control

## 4. Time Tracking

### 4.1 Why We Track Time

- Accurate project costing (are we profitable on each project?)
- Capacity planning (who has bandwidth for new work?)
- Estimation improvement (compare estimates vs. actuals)
- Client billing (for retainer and hourly clients)
- Team workload balance (prevent burnout)

### 4.2 Tracking Rules

| Rule | Standard |
| :--- | :--- |
| Tool | Toggl Track (all team members) |
| Granularity | Minimum 15-minute increments |
| Categories | Client work, internal projects, admin, meetings, learning |
| When to track | Real-time (start/stop timer), not end-of-day reconstruction |
| Minimum billable hours | 6 hours/day (out of 8 hours present) |
| Non-billable allowance | 2 hours/day (admin, meetings, breaks, learning) |
| Weekly review | Studio Coordinator reviews all timesheets every Monday |

### 4.3 Time Codes

| Code | Category | Examples |
| :--- | :--- | :--- |
| [CLIENT]-DESIGN | Billable design work | Logo creation, layout design, illustration |
| [CLIENT]-REVISION | Billable revisions | Client-requested changes within scope |
| [CLIENT]-MEETING | Billable meetings | Client calls, presentations, reviews |
| [CLIENT]-ADMIN | Non-billable client admin | File organization, email, coordination |
| INT-MARKETING | Internal marketing | Studio social media, portfolio updates |
| INT-ADMIN | Internal admin | Team meetings, HR, finance, operations |

## 5. Capacity Planning

### 5.1 Weekly Capacity Model

| Role | Available Hours/Week | Target Billable | Buffer |
| :--- | :--- | :--- | :--- |
| Senior Designer | 40 | 30 (75%) | 10 (meetings, mentoring, reviews) |
| Mid-Level Designer | 40 | 32 (80%) | 8 (learning, admin) |
| Junior Designer | 40 | 34 (85%) | 6 (learning, admin) |
| Account Manager | 40 | 20 (50%) | 20 (client management, new business) |
| Creative Director | 40 | 16 (40%) | 24 (strategy, reviews, leadership) |

### 5.2 Capacity Dashboard (Updated Weekly)

| Team Member | This Week Allocated | Next Week Allocated | Available Hours | Status |
| :--- | :--- | :--- | :--- | :--- |
| Designer A | 32/32 | 28/32 | 4 next week | 🔴 Full |
| Designer B | 24/32 | 20/32 | 12 next week | 🟡 Available |
| Designer C | 30/34 | 34/34 | 0 next week | 🔴 Full |
| Junior D | 20/34 | 15/34 | 19 next week | 🟢 Open |

### 5.3 Overload Protocol

When a team member exceeds 90% allocation for 2+ consecutive weeks:
1. Studio Coordinator flags to Creative Director
2. Options assessed: redistribute work, extend timeline, bring in freelancer
3. Decision made within 24 hours
4. Team member informed and workload adjusted
5. If pattern persists: hiring discussion initiated
