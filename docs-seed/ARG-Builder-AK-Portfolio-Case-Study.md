# ArtKech Design Studio — Portfolio & Case Study Development

## 1. Purpose

The portfolio is ArtKech's primary sales tool. Every potential client evaluates us through our published work before making contact. This SOP defines how we select, document, photograph, write, and publish case studies — transforming completed projects into compelling narratives that attract ideal clients and justify premium pricing.

## 2. Case Study Selection Criteria

### 2.1 Scoring Matrix (Score Each Project 1-5)

| Criterion | Weight | Question to Ask |
| :--- | :--- | :--- |
| Visual impact | 25% | Does this work look stunning in a portfolio context? |
| Strategic depth | 20% | Can we tell a meaningful story about the thinking behind it? |
| Client prestige | 15% | Does this client name add credibility? |
| Industry diversity | 15% | Does this fill a gap in our portfolio (new sector)? |
| Results/impact | 15% | Can we show measurable business outcomes? |
| Recency | 10% | Is this recent enough to feel current? |

### 2.2 Selection Rules

| Rule | Standard |
| :--- | :--- |
| Minimum score to publish | 3.5/5.0 average |
| Portfolio size (website) | 12-18 featured case studies |
| Refresh frequency | Add 2-3 new cases per quarter, retire oldest/weakest |
| Industry balance | No more than 3 cases from same industry |
| Service balance | Represent all core services (branding, print, digital, social) |
| Client permission | Written approval required before publishing |

## 3. Case Study Structure

### 3.1 Standard Format

| Section | Content | Length |
| :--- | :--- | :--- |
| Hero image | Best single visual from the project | Full-width, high-impact |
| Title | Client name + project type | 5-8 words |
| Tagline | One-sentence summary of what we did | 10-15 words |
| Overview | Client context, challenge, our role | 3-4 sentences |
| Challenge | What problem needed solving | 2-3 sentences |
| Approach | Our strategic and creative process | 3-5 sentences |
| Solution | What we created and why | 3-5 sentences |
| Results | Measurable outcomes (if available) | 2-4 bullet points |
| Visuals | 8-15 images showing the work in context | Full gallery |
| Credits | Team members, photographers, collaborators | Short list |

### 3.2 Writing Guidelines

| Element | Standard |
| :--- | :--- |
| Voice | Confident but not arrogant, strategic, specific |
| Tense | Past tense for the project, present tense for the brand |
| Length | 200-400 words total (people scan, not read) |
| Jargon | Avoid design jargon — write for business owners |
| Results | Use numbers whenever possible ("increased foot traffic by 40%") |
| Client quotes | Include if available (adds credibility) |

### 3.3 Example Structure

> **Riad Dar Zellij — Brand Identity & Collateral**
>
> *Crafting a luxury hospitality brand rooted in Moroccan artisanship*
>
> **Overview:** Riad Dar Zellij is a boutique riad in Marrakech's medina offering curated cultural experiences. They needed a brand identity that communicated luxury, authenticity, and the warmth of Moroccan hospitality.
>
> **Challenge:** Compete with international hotel brands while maintaining the intimate, artisanal character that makes riads special. The previous identity felt generic and didn't reflect the property's unique zellige tilework heritage.
>
> **Approach:** We immersed ourselves in the riad's story — its 18th-century architecture, the master craftsmen who restored it, and the guests who seek authentic cultural connection. We developed a visual language inspired by geometric zellige patterns, translated into a modern luxury context.
>
> **Solution:** A comprehensive brand identity built around a custom geometric mark derived from traditional zellige patterns. The system extends across stationery, room collateral, signage, digital presence, and guest experience touchpoints.
>
> **Results:**
> - 35% increase in direct bookings within 6 months of rebrand
> - Featured in Condé Nast Traveler's "Best New Hotel Branding"
> - Guest satisfaction scores improved from 4.2 to 4.7/5.0

## 4. Photography & Visual Documentation

### 4.1 What to Photograph

| Category | Examples | Priority |
| :--- | :--- | :--- |
| Hero shots | Best single image, maximum impact | Essential |
| In-context | Logo on building, menu on table, card in hand | Essential |
| Detail shots | Close-ups of typography, texture, finishing | Important |
| Process | Sketches, mood boards, explorations | Nice to have |
| Flat lays | Collateral arranged artistically | Important |
| Environment | The space/business where the brand lives | Important |
| Digital screens | Website/app on devices (mockup or real) | If applicable |

### 4.2 Photography Standards

| Element | Standard |
| :--- | :--- |
| Camera | DSLR or mirrorless (minimum 24MP) |
| Lighting | Natural light preferred, or professional studio lighting |
| Background | Clean, contextual, or branded (never cluttered) |
| Props | Minimal, relevant, high-quality (fresh flowers, quality paper, clean surfaces) |
| Editing | Consistent color grading across all images in a case study |
| Resolution | Minimum 3000px on longest edge for web portfolio |
| Format | RAW capture, deliver as JPG (quality 90%) + original RAW |

### 4.3 Mockup Standards

| Rule | Standard |
| :--- | :--- |
| Quality | Premium mockups only (no free low-quality ones) |
| Realism | Perspective, shadows, and lighting must look natural |
| Consistency | Same mockup style across a single case study |
| Variety | Mix real photography with mockups (never all mockups) |
| Sources | Preferred: Yellow Images, Mr. Mockup, Original Mockups |

## 5. Client Permission Process

### 5.1 Permission Request Template

> Subject: Permission to Feature [Project] in Our Portfolio
>
> Hi [Name],
>
> We're incredibly proud of the work we created together on [project], and we'd love to feature it in our portfolio and case studies.
>
> This would include:
> - Images of the deliverables on our website portfolio
> - A brief case study describing the project (we'll share for your approval before publishing)
> - Potential inclusion in our social media and awards submissions
>
> Of course, we'll share the final case study with you for approval before anything goes live. You'll have full veto power over any content or images.
>
> Would you be comfortable with this? If there are any restrictions (e.g., no financial results, certain images excluded), we're happy to accommodate.
>
> Thank you for the wonderful collaboration!

### 5.2 Permission Levels

| Level | What We Can Share | Common For |
| :--- | :--- | :--- |
| Full permission | Everything including results and client quotes | Most clients |
| Partial permission | Visuals only, no financial data or strategy details | Corporate clients |
| Anonymous | Work shown but client name withheld | Sensitive industries |
| Embargoed | Can publish after a specific date | Pre-launch projects |
| Denied | Cannot show publicly | Rare (respect immediately) |
