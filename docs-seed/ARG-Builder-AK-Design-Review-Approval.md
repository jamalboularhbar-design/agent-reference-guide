# ArtKech Design Studio — Design Review & Approval Workflow

## 1. Purpose

This SOP defines the internal and external review process for all creative work produced at ArtKech Design Studio. A structured review workflow prevents subjective feedback loops, protects creative quality, and ensures client satisfaction within the agreed scope and timeline.

## 2. Review Stages Overview

Every project passes through 4 review gates before final delivery:

| Stage | Reviewer | Purpose | Duration |
| :--- | :--- | :--- | :--- |
| 1. Self-review | Designer | Technical quality, brief alignment, file hygiene | 30 min |
| 2. Internal review | Creative Director | Creative quality, brand consistency, strategic alignment | 1-2 hours |
| 3. Client presentation | Client | Direction approval, feedback collection | 2-3 business days |
| 4. Final QA | Production Manager | Print/digital specs, file formats, delivery readiness | 1-2 hours |

## 3. Stage 1: Self-Review Checklist

Before submitting work for internal review, the designer verifies:

### 3.1 Technical Quality

| Check | Standard |
| :--- | :--- |
| Resolution | Print: 300 DPI minimum. Digital: 72 DPI at target size |
| Color mode | Print: CMYK. Digital: sRGB. Logo: both versions exist |
| Bleed | Print files: 3mm bleed on all sides |
| Typography | All text is editable (not rasterized), fonts outlined in final files only |
| Alignment | Grid-based layout, consistent spacing, optical alignment verified |
| Spelling | All text proofread (Arabic and French checked by native speaker) |

### 3.2 Brief Alignment

| Check | Standard |
| :--- | :--- |
| Deliverables match brief | Every item listed in brief is produced |
| Brand personality reflected | Design embodies the 3-5 adjectives from brief |
| Target audience appropriate | Visual language resonates with defined audience |
| Constraints respected | No forbidden colors, styles, or elements used |
| Inspiration references considered | Design shows awareness of references without copying |

### 3.3 File Organization

| Check | Standard |
| :--- | :--- |
| Layer naming | Descriptive names (not "Layer 1", "Copy of Group") |
| Artboard naming | Matches deliverable name from brief |
| Working file saved | .ai / .psd / .fig saved in project folder |
| Presentation mockup | At least 2 realistic mockups prepared for client presentation |

## 4. Stage 2: Internal Review

### 4.1 Review Session Format

| Element | Detail |
| :--- | :--- |
| Duration | 30-60 minutes per project |
| Attendees | Creative Director + Lead Designer + Account Manager (optional) |
| Format | Screen share or printed proofs, designer presents rationale first |
| Feedback method | Creative Director gives verbal feedback, designer takes notes |
| Documentation | Feedback summarized in Notion project page within 1 hour |

### 4.2 Creative Director Evaluation Criteria

| Criteria | Weight | What's Assessed |
| :--- | :--- | :--- |
| Concept strength | 30% | Is there a clear idea? Does it solve the brief's problem? |
| Visual execution | 25% | Craft quality, typography, composition, color harmony |
| Brand fit | 20% | Does it feel right for the client's audience and industry? |
| Originality | 15% | Is it distinctive? Would it stand out in the competitive landscape? |
| Scalability | 10% | Will it work across all required applications and sizes? |

### 4.3 Feedback Categories

| Category | Meaning | Action Required |
| :--- | :--- | :--- |
| Approved | Ready for client presentation | Proceed to mockup and presentation prep |
| Minor revisions | Small adjustments needed (color tweak, spacing, type size) | Fix within 2 hours, no re-review needed |
| Major revisions | Significant changes needed (layout, concept direction) | Revise and re-submit for internal review |
| Restart | Concept doesn't work, new direction needed | Schedule creative brainstorm, new concept within 2 days |

## 5. Stage 3: Client Presentation

### 5.1 Presentation Preparation

| Element | Standard |
| :--- | :--- |
| Presentation format | PDF deck (branded ArtKech template) or Figma prototype |
| Number of concepts | 2-3 distinct directions (never just 1, never more than 3) |
| Mockups per concept | Minimum 2 realistic applications (e.g., business card + signage) |
| Rationale | Each concept includes 2-3 sentences explaining the thinking |
| Mood board reference | Show how the concept connects to approved inspiration |

### 5.2 Presentation Delivery

| Method | When to Use | Notes |
| :--- | :--- | :--- |
| In-person at studio | Projects > 30,000 MAD, brand identities, VIP clients | Preferred — allows real-time discussion |
| Video call (Zoom/Meet) | Remote clients, international projects | Screen share + PDF sent after |
| Email (PDF) | Simple projects < 10,000 MAD, repeat clients who prefer async | Include cover note explaining each concept |

### 5.3 Managing Client Feedback

**Rules for receiving feedback:**
- Never present without context — always explain the "why" before showing the "what"
- Ask the client to react to the concept/direction first, details second
- If client says "I don't like it" — ask "What specifically isn't working? Is it the color, the style, the feeling?"
- Document all feedback verbatim during the meeting (Account Manager's responsibility)
- Summarize feedback in writing and send to client within 24 hours for confirmation

**Revision rounds included:**
| Project Value | Included Rounds | Additional Round Cost |
| :--- | :--- | :--- |
| < 10,000 MAD | 2 rounds | 1,500 MAD per round |
| 10,000 – 30,000 MAD | 2 rounds | 2,500 MAD per round |
| 30,000 – 80,000 MAD | 3 rounds | 3,500 MAD per round |
| > 80,000 MAD | 3 rounds | 5,000 MAD per round |

**What counts as a "round":**
- A round = one consolidated set of feedback, submitted within the agreed review period
- Multiple emails with piecemeal feedback within the same review window = 1 round
- New feedback after approval of a previous round = new round
- Scope changes (new deliverables, new direction) = not a revision, requires new quote

## 6. Stage 4: Final QA

### 6.1 Production Checklist

| Check | Print Files | Digital Files |
| :--- | :--- | :--- |
| Color mode | CMYK (Pantone references noted) | sRGB |
| Resolution | 300 DPI minimum | 72 DPI at target pixel dimensions |
| Bleed & trim | 3mm bleed, trim marks included | N/A |
| File format | PDF/X-4 (press-ready) + editable source | PNG, JPG, SVG, PDF as specified |
| Font handling | Outlined in final PDF, editable in source file | Embedded or web-safe |
| Image links | All linked images embedded or packaged | Optimized for web (< 500KB per image) |
| Naming convention | [Client]_[Deliverable]_[Version]_FINAL.pdf | [Client]_[Deliverable]_[Platform]_[Size].png |

### 6.2 Delivery Package Structure

```
📁 [Client Name] — Final Delivery
├── 📁 Print Files
│   ├── 📁 Business Cards (PDF/X-4 + AI source)
│   ├── 📁 Letterhead (PDF/X-4 + AI source)
│   └── 📁 [Other print deliverables]
├── 📁 Digital Files
│   ├── 📁 Social Media Templates (PSD/Figma + exported PNG)
│   ├── 📁 Website Assets (SVG + PNG at 1x, 2x)
│   └── 📁 Email Signature (HTML + PNG)
├── 📁 Logo Package
│   ├── Primary Logo (AI, SVG, PNG, JPG — color + mono + reversed)
│   ├── Secondary Logo (same formats)
│   └── Favicon / App Icon (PNG 512x512, 192x192, 32x32)
├── 📁 Brand Guidelines (if included)
│   └── Brand_Guidelines_v1.0.pdf
└── 📄 README.txt (file overview, font list, color codes)
```

## 7. Approval Documentation

### 7.1 Sign-Off Process

- Final work is presented to client with a sign-off request
- Client confirms approval via email (written confirmation required)
- Sign-off email is saved in project folder as proof of approval
- After sign-off: any changes are billed as new work (not revisions)

### 7.2 Sign-Off Template

> Subject: [Project Name] — Final Approval Request
>
> Dear [Client Name],
>
> Please find attached the final deliverables for [project name]. All revisions from our last review have been incorporated.
>
> Could you please confirm your approval by replying to this email with "Approved" or noting any final adjustments?
>
> Once approved, we will prepare the production files and deliver the complete package within 2 business days.
>
> Thank you for a wonderful collaboration!
