# ArtKech Design Studio — Client Onboarding & Project Kickoff

## 1. Purpose

This SOP covers the end-to-end process from the moment a client signs a contract to the internal kickoff that launches creative work. Proper onboarding sets the tone for the entire relationship — it demonstrates professionalism, builds trust, and prevents misunderstandings that derail projects later.

## 2. Onboarding Timeline

| Day | Milestone | Owner | Client Action Required |
| :--- | :--- | :--- | :--- |
| Day 0 | Contract signed + deposit received | Account Manager | Sign contract, pay 50% deposit |
| Day 0-1 | Welcome email sent | Account Manager | Read and acknowledge |
| Day 1 | Client folder created (Drive + Notion) | Studio Coordinator | — |
| Day 1-2 | Asset collection request sent | Account Manager | Provide logos, photos, copy, brand guide |
| Day 2-3 | Assets received and organized | Studio Coordinator | Upload/send all materials |
| Day 3 | Internal kickoff meeting | Creative Director | — |
| Day 3-4 | Creative work begins | Lead Designer | — |
| Day 5 | Progress check-in (if project > 2 weeks) | Account Manager | Brief status update to client |

## 3. Welcome Package

### 3.1 Welcome Email Contents

Sent within 4 hours of deposit confirmation:

| Element | Content |
| :--- | :--- |
| Subject line | "Welcome to ArtKech — Let's create something remarkable 🎨" |
| Greeting | Personal, warm, references the project specifically |
| What happens next | Clear 5-step timeline of what to expect |
| Asset request | Specific list of materials needed from client (see Section 4) |
| Communication preferences | Ask: preferred channel (email/WhatsApp/call), response time expectations |
| Key contacts | Account Manager name + direct line, Studio Coordinator for logistics |
| Project portal link | Notion page where client can track progress (view-only) |
| Attachment | ArtKech Process Guide (1-page PDF explaining our workflow) |

### 3.2 Process Guide (1-page PDF)

A branded one-pager explaining:
- Our 4-stage process: Brief → Concepts → Refinement → Delivery
- What "revision rounds" mean and how many are included
- Expected response times (ours and theirs)
- How to give effective feedback (specific > vague, objective > subjective)
- Payment schedule reminder

## 4. Asset Collection

### 4.1 Standard Asset Request

| Asset | Format Preferred | Required For |
| :--- | :--- | :--- |
| Existing logo (if any) | AI, SVG, or high-res PNG | Reference, evolution, or integration |
| Brand guidelines (if any) | PDF | Color, typography, tone constraints |
| Product/team photos | High-res JPG (minimum 2000px wide) | Mockups, layouts, website |
| Copy/text content | Word doc or Google Doc | Layouts, typesetting |
| Competitor examples | URLs or screenshots | Competitive positioning |
| Inspiration references | Pinterest board, screenshots, or URLs | Creative direction |
| Technical specs | Dimensions, materials, print method | Production accuracy |

### 4.2 Asset Follow-Up Protocol

| Day | Action |
| :--- | :--- |
| Day 2 | Initial request sent (in welcome email) |
| Day 4 | Gentle reminder if not received: "Just checking in — do you need help gathering these?" |
| Day 6 | Escalation: Account Manager calls client directly |
| Day 8 | Timeline adjustment: "Without these assets, we'll need to push the first milestone by X days" |

**Rule:** Creative work does not begin until minimum viable assets are received. Waiting for assets is the #1 cause of timeline delays — we communicate this clearly upfront.

## 5. Client Communication Setup

### 5.1 Channel Selection

| Project Value | Primary Channel | Secondary Channel | Meeting Frequency |
| :--- | :--- | :--- | :--- |
| < 10,000 MAD | WhatsApp | Email (for files) | As needed (async) |
| 10,000 – 30,000 MAD | Email | WhatsApp (quick questions) | Bi-weekly check-in |
| 30,000 – 80,000 MAD | Email + Notion | WhatsApp + scheduled calls | Weekly 15-min call |
| > 80,000 MAD | Dedicated Slack channel | Email (formal) + calls | Weekly 30-min call |

### 5.2 Communication Rules

- All project decisions must be documented in writing (email or Notion comment)
- WhatsApp is for quick questions and logistics — never for approvals or scope changes
- Client receives a progress update minimum once per week (even if it's "on track, no news")
- Account Manager is the single point of contact — designers do not communicate with clients directly (except during presentations)

## 6. Internal Kickoff Meeting

### 6.1 Attendees & Roles

| Role | Responsibility in Kickoff |
| :--- | :--- |
| Account Manager | Presents brief, shares client context, answers questions |
| Creative Director | Sets creative direction, assigns team, identifies risks |
| Lead Designer | Asks questions, confirms understanding, proposes initial approach |
| Studio Coordinator | Confirms timeline, identifies resource conflicts, sets up project tracking |
| Junior Designer (if assigned) | Listens, takes notes, asks clarifying questions |

### 6.2 Kickoff Agenda (30 minutes)

| Block | Duration | Content |
| :--- | :--- | :--- |
| Brief walkthrough | 10 min | Account Manager presents the approved brief |
| Client context | 5 min | Who is this client? What's their personality? What do they value? |
| Creative direction | 10 min | Creative Director shares initial thoughts, references, constraints |
| Timeline & logistics | 5 min | Studio Coordinator confirms milestones, assigns tasks in Notion |

### 6.3 Kickoff Outputs

- [ ] Lead Designer assigned and confirmed
- [ ] Notion project board populated with tasks and deadlines
- [ ] Time tracking code created and shared with team
- [ ] Client folder organized with all received assets
- [ ] First internal milestone set (usually "mood board" or "initial concepts")
- [ ] Any blockers identified and assigned for resolution

## 7. Project Setup in Notion

### 7.1 Project Page Structure

```
📄 [Client] — [Project Name]
├── Status: 🟡 In Progress
├── Account Manager: [Name]
├── Lead Designer: [Name]
├── Start Date: [Date]
├── Deadline: [Date]
├── Budget: [Amount] MAD
├── 📋 Brief (embedded)
├── 📋 Timeline & Milestones
│   ├── ☐ Mood board / research (Day 3)
│   ├── ☐ Concept development (Day 5)
│   ├── ☐ Internal review (Day 6)
│   ├── ☐ Client presentation (Day 7)
│   ├── ☐ Revisions (Day 9-11)
│   ├── ☐ Final QA (Day 12)
│   └── ☐ Delivery (Day 13)
├── 📋 Feedback Log
├── 📋 Time Tracking Summary
└── 📁 Files (linked to Drive folder)
```

### 7.2 Client View (Read-Only)

Clients with Notion access see:
- Current project status (In Progress / Awaiting Feedback / Delivered)
- Next milestone and expected date
- Files shared for review
- Comment thread for feedback

They do NOT see: internal notes, time tracking, cost breakdowns, or team discussions.

## 8. Onboarding for Retainer Clients

For clients on monthly retainers (ongoing design support):

| Element | Standard |
| :--- | :--- |
| Onboarding meeting | 1-hour strategy session (brand deep-dive, content calendar review) |
| Dedicated team | Same designer assigned for consistency |
| Communication | Weekly 15-min check-in + dedicated WhatsApp group |
| Request process | Submit via Notion form or email, response within 4 hours |
| Hours tracking | Monthly report showing hours used vs allocated |
| Rollover policy | Unused hours do NOT roll over (use it or lose it) |
| Scope | Defined in retainer agreement (e.g., "up to 20 social posts + 2 print pieces per month") |

## 9. Performance Metrics

| KPI | Target | Measurement |
| :--- | :--- | :--- |
| Onboarding completion (all steps done) | < 3 business days | Project management system |
| Asset collection time | < 5 business days from request | Tracking in Notion |
| Client satisfaction with onboarding | > 4.5/5.0 | Post-project survey |
| Internal kickoff held within | 1 business day of asset receipt | Calendar tracking |
| Project start delay (due to missing assets) | < 10% of projects | Monthly review |
